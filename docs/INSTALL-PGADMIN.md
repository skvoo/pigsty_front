# Установка PgAdmin через Pigsty

PgAdmin в конфиге уже описан (`pigsty.yml` → группа `app`, приложение `pgadmin`). Чтобы его поставить, плейбуки нужно выполнять **из каталога полной установки Pigsty** на сервере (где лежат `app.yml`, `docker.yml`, `configure` и т.д.).

## Шаги

1. **Подключиться к серверу** (или с машины, откуда запускаете Ansible):

   ```bash
   ssh st@104.223.25.234
   ```

2. **Перейти в каталог Pigsty** (путь может быть другим, например `~/pigsty` или `/opt/pigsty`):

   ```bash
   cd ~/pigsty   # или где у вас развёрнут Pigsty
   ```

3. **Убедиться, что инвентарь указывает на ваш конфиг** (если конфиг в другом месте, передать через `-e` или скопировать как `pigsty.yml` в этот каталог).

4. **Включить Docker на хостах группы `app`** (если ещё не делали):

   ```bash
   ./docker.yml -l app
   ```

5. **Развернуть приложения** (в т.ч. pgadmin):

   ```bash
   ./app.yml -l app
   ```

   В конфиге по умолчанию `app: pgadmin`, поэтому поднимется именно PgAdmin.

## После установки

- **URL:** http://104.223.25.234:8885  
- **Логин:** `admin@pigsty.cc`  
- **Пароль:** `pigsty` (задан в `pigsty.yml` → `apps.pgadmin.conf.PGADMIN_DEFAULT_*`)

Рекомендуется сменить пароль после первого входа (в PgAdmin: File → Preferences → или смена в конфиге и перезапуск контейнера).

## Добавление сервера PostgreSQL в PgAdmin

1. Правый клик по **Servers** → **Register** → **Server**.
2. **General:** имя, например `pg-meta`.
3. **Connection:**
   - **Host:** `104.223.25.234` (или `127.0.0.1`, если PgAdmin запущен на том же сервере).
   - **Port:** `5432` (напрямую к Postgres) или `6432` (PgBouncer).
   - **Username:** например `dbuser_meta` или `supabase_admin`.
   - **Password:** из `pg_users` в `pigsty.yml`.

После сохранения в дереве появятся все базы кластера (meta, app, supabase и т.д.).

## Страница входа крутится и не загружается (бесконечный спиннер)

Частые причины и что сделать:

1. **Старая сессия после перезапуска контейнера**  
   Закройте **все** вкладки с pgAdmin, очистите cookies для `http://104.223.25.234:8885` или откройте ссылку в **режиме инкогнито** и попробуйте снова. После рестарта контейнера старые CSRF-токены недействительны, из-за этого фронт «висит» на загрузке.

2. **Проверить, что бэкенд отвечает** (на сервере или с машины, откуда ходите в браузер):
   ```bash
   curl -s -o /dev/null -w "%{http_code}" http://104.223.25.234:8885/
   ```
   Ожидается `200` или `302`. Если таймаут или 5xx — смотреть логи контейнера.

3. **Логи контейнера pgAdmin** (на сервере):
   ```bash
   docker ps | grep pgadmin
   docker logs <имя_или_id_контейнера> --tail 100
   ```
   Ошибки Python/Flask или Gunicorn подскажут причину.

4. **Доступ по IP и защита cookies**  
   Если спиннер не исчезает даже в инкогнито, можно ослабить защиту cookie (только если доступ по IP без HTTPS приемлем в вашей среде). В `pigsty.yml` в `apps.pgadmin.conf` добавить:
   ```yaml
   PGADMIN_CONFIG_ENHANCED_COOKIE_PROTECTION: 'False'
   ```
   Затем пересоздать контейнер: `./app.yml -l app` (или перезапуск контейнера pgadmin после изменения `.env` в `/opt/pgadmin` на сервере).

5. **Кэш браузера**  
   Жёсткое обновление: Ctrl+F5 (или Cmd+Shift+R на Mac).

## Если в каталоге Pigsty нет `app.yml`

Значит, используется не полный дистрибутив. Варианты:

- Скачать полный Pigsty с [pigsty.io](https://pigsty.io) и развернуть из него, используя ваш `pigsty.yml` как инвентарь.
- Поставить PgAdmin вручную через Docker на том же сервере (образ `dpage/pgadmin4`, порт 8885, переменные `PGADMIN_DEFAULT_EMAIL` и `PGADMIN_DEFAULT_PASSWORD`).
