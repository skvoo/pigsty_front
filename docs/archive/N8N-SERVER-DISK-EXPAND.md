# Расширение корневого раздела на сервере n8n (172.245.64.199)

> **Архив:** документ относится к серверу **172.245.64.199** (n8n), не к Pigsty 104.223.25.234. Перенесён в docs/archive/ 16.02.2026.

**Сервер:** 172.245.64.199 (root)  
**Задача:** расширить корневой раздел после добавления 50 ГБ диска.

---

## Солюшн: пошаговый план (делать по порядку)

Ниже один линейный сценарий: от подключения до проверки и защиты от повторного заполнения. Команды копируй и выполняй по шагам.

### Перед началом
- Уже докупили 50 ГБ в панели хостинга.
- Перезагрузил VPS **из панели** (не из ОС), чтобы диск увидел новый размер.
- По возможности зайти в **screen** или **tmux**, чтобы при обрыве SSH команды не оборвались.

---

### Шаг 1. Подключиться и проверить диск

```bash
ssh root@172.245.64.199
lsblk
```

В колонке **SIZE** смотри строку диска (`vda` или `sda` без цифры). Должно быть ~100 ГБ или больше (старый размер + 50). Если по-прежнему старый размер — перезагрузка только из панели, затем снова `lsblk`.

**Если у тебя так:** `vda 150G`, `vda1 97.8G /`, `vda2 2.3G [SWAP]` — свободные ~50 ГБ идут **после** swap (диск вырос с конца). Команду `resizepart 1 100%` **не использовать**: она затрёт swap. Нужен **Способ B (fdisk)** — готовые команды в блоке ниже («Твой случай: 150G, root + swap»).

---

### Шаг 2. Parted — расширить раздел (только если свободное место между vda1 и vda2)

```bash
parted /dev/vda
```

В интерактиве:
```
print
```
Убедись: диск вырос, есть раздел 1 (root) и 2 (swap).

```
resizepart 1 100%
```
Если спросит *"Partition is being used. Are you sure?"* → **Yes**.

```
quit
```

---

### Шаг 3. Расширить файловую систему

Если корень на **XFS** (как на этом сервере):
```bash
xfs_growfs /
```

Если бы был **ext4**:
```bash
resize2fs /dev/vda1
```

---

### Шаг 4. Проверка

```bash
df -h
```

Раздел `/` должен показать новый размер (порядка 145–148 ГБ). После этого система и SSH обычно приходят в норму.

---

### Шаг 5. Ограничить логи Docker (обязательно)

Иначе диск снова забьётся.

```bash
mkdir -p /etc/docker
nano /etc/docker/daemon.json
```

Вставить (или дописать к существующему):
```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "50m",
    "max-file": "2"
  }
}
```

Сохранить, затем:
```bash
systemctl restart docker
```

---

### Шаг 6. Меньше сохранять выполнений n8n

Найти, где у n8n заданы переменные (`.env` или `systemd`/docker), и добавить/выставить:

```
EXECUTIONS_DATA_SAVE_ON_SUCCESS=false
EXECUTIONS_DATA_SAVE_ON_ERROR=false
```

Перезапустить n8n (например `systemctl restart n8n` или перезапуск контейнера).

---

**Готово.** Диск расширен, место освободилось, повторное заполнение ограничено логами Docker и сохранением выполнений n8n.

Если при `resizepart` layout окажется другим (например, нет места между разделами) — ниже в документе описан **Способ B** через fdisk.

---

### Твой случай: vda 150G, vda1 97.8G, vda2 2.3G (свободное место после swap)

Раз свободные 50 ГБ после swap, делаем через fdisk: временно убираем swap, расширяем root, заново создаём swap. **Перед этим зайди в screen/tmux.**

---

#### Проверка безопасности: ничего не затрём

| Что на диске | Что делаем | Риск затереть? |
|--------------|------------|-----------------|
| **Данные корня /** (0 … ~98 ГБ) | В fdisk «удаляем» раздел 1 — стирается только **запись в таблице разделов**. Байты на диске не трогаются. Потом создаём раздел 1 заново с **тем же началом (сектор 2048)** и размером +147G. Старые данные остаются на месте; добавляется только свободная зона справа. | **Нет**, если не менять первый сектор и не удалять подпись (см. ниже). |
| **Swap (vda2)** | Удаляем и создаём заново. В swap нет постоянных данных, только кэш ОЗУ. | **Нет** (swap не хранит то, что нельзя потерять). |
| **Подпись ФС (XFS)** в начале vda1 | fdisk может спросить: *«Remove signature?»* — это про первые сектора раздела. Если ответить **Y**, будет затерта метаданные XFS и данные станут нечитаемы. | **Да, если ответить Y.** Всегда отвечать **N**. |

**Критичные моменты (сверить перед действием):**

1. **Первый сектор раздела 1** — при создании раздела 1 на вопрос *«First sector»* нажимать **только Enter** (оставить по умолчанию **2048**). Не вводить другое число — иначе раздел сдвинется и текущие данные корня окажутся «вне» раздела или будут перезаписаны.
2. **Remove signature?** — ответ только **N** (No). Подпись XFS в начале раздела должна остаться.
3. **Между `w` в fdisk и `xfs_growfs /`** — не перезагружать и не выключать сервер. Сразу выполнить `partprobe` и `xfs_growfs /`.
4. **Файловая система** — у тебя корень на **XFS**. Использовать именно `xfs_growfs /`. Команда `resize2fs` только для ext4; для XFS не подходит.

**Проверить тип ФС перед расширением (по желанию):**
```bash
findmnt / -o FSTYPE
# или
lsblk -f /
```
Должно быть **xfs**.

---

**1. Отключить swap**
```bash
swapoff -a
```

**2. fdisk**
```bash
fdisk /dev/vda
```

В интерактиве по очереди:
- `d` → Enter (удалить раздел 2 — swap)
- `d` → Enter (удалить раздел 1 — root; данные не стираются, меняется только таблица разделов)
- `n` → `p` → `1` → **Enter** (First sector — обязательно по умолчанию, обычно 2048; **не вводить другое число**) → **`+147G`** (под корень 147 ГБ, под swap останется ~3 ГБ)
- На вопрос *"Remove signature? [Y]es/[N]o"* → только **`N`** (иначе затрётся начало ФС и данные)
- `n` → `p` → `2` → Enter → Enter (остаток под swap)
- `t` → `2` → `82` (тип Linux swap)
- Перед `w`: ещё раз проверить — раздел 1 начинается с 2048 (или что предложил fdisk по умолчанию), размер +147G; на вопрос про signature ответили **N**.
- `w` (записать и выйти)

**3. Сразу после выхода из fdisk**
```bash
partprobe
xfs_growfs /
```
(Если ФС ext4 — `resize2fs /dev/vda1`.)

**4. Включить swap**
```bash
mkswap /dev/vda2
swapon -a
```

**5. Проверка**
```bash
df -h
free -m
```
