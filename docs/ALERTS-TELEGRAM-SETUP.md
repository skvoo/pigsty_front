# Включение алертов и вывод в Telegram

## 1. Как включаются алерты

Алерты уже **включены**: VMAlert на Pigsty по правилам из `/infra/rules/*.yml` (node, pgsql и др.) вычисляет условия и отправляет срабатывания в Alertmanager. Сейчас в Alertmanager стоит receiver `default`, который ничего не делает — уведомления никуда не уходят.

Чтобы алерты **реально доходили**, нужно:
- добавить в Alertmanager **receiver** с доставкой (Telegram, email и т.д.);
- настроить **route**, чтобы нужные алерты шли в этот receiver.

Итог: правила не трогаем, меняем только конфиг Alertmanager.

---

## 2. Telegram: бот или канал?

Нужны **оба**, но по-разному:

- **Бот** — обязателен. Alertmanager шлёт сообщения через Telegram Bot API по токену бота. Без бота в Telegram оповещения не отправить.
- **Куда слать** — на выбор:
  - **Личный чат с ботом** — удобно для одного человека. Создаёте бота, пишете ему любое сообщение, получаете `chat_id` этого чата.
  - **Группа** — бота добавляете в группу, сообщения приходят в группу; `chat_id` будет вида `-123456789` или `-100xxxxxxxxxx`.
  - **Канал** — бота добавляете в канал как администратора (с правом публиковать сообщения). У канала свой `chat_id` (обычно вида `-100xxxxxxxxxx`).

Рекомендация: для алертов чаще делают **отдельного бота** (например `@YourAlertsBot`) и либо личный чат с ним, либо **отдельный канал** типа «Alerts» — тогда все уведомления в одном месте и не мешают в общих чатах. Отдельный канал не обязателен: можно слать и в личку боту, и в общую группу.

## 3. Пошагово: бот и chat_id

### Шаг 1: Создать бота

1. В Telegram откройте [@BotFather](https://t.me/BotFather).
2. Отправьте `/newbot`.
3. Введите имя бота (например `Pigsty Alerts`) и username (например `YourPigstyAlertsBot`).
4. BotFather пришлёт **токен** вида `123456789:ABCdefGHI...` — это **bot_token**. Сохраните его, никому не показывайте.

### Шаг 2: Узнать chat_id

**Вариант A: личный чат с ботом**

1. Найдите бота по username и нажмите «Start» / напишите любое сообщение.
2. В браузере откройте (подставьте свой токен):
   ```
   https://api.telegram.org/bot<BOT_TOKEN>/getUpdates
   ```
3. В ответе в `message.chat.id` будет число, например `987654321` — это **chat_id**.

**Вариант B: канал**

1. Создайте канал (например «Alerts»).
2. Добавьте бота в канал как администратора с правом «Post messages».
3. Опубликуйте в канале любое сообщение (может написать сам бот или вы от его имени).
4. Снова откройте `getUpdates` — в ответе будет `chat.id` канала, обычно вида `-1001234567890`. В конфиге **chat_id обязательно в кавычках** (строка), например `"-1001234567890"`.

**Вариант C: группа**

1. Добавьте бота в группу.
2. Напишите в группе что угодно.
3. По `getUpdates` возьмите `message.chat.id` группы (отрицательное число).

## 4. Настройка Alertmanager на сервере Pigsty

Конфиг: **`/etc/alertmanager.yml`**. Редактировать можно на сервере (от root или через sudo).

### 4.1 Добавить receiver «telegram»

В секции `receivers:` раскомментируйте и заполните блок Telegram (уже есть в файле как пример):

```yaml
  - name: 'telegram'
    telegram_configs:
      - api_url: 'https://api.telegram.org'
        bot_token: 'ВАШ_ТОКЕН_ОТ_BOTFATHER'
        chat_id: 'ВАШ_CHAT_ID'   # строка! для канала/группы: "-1001234567890"
        send_resolved: true
        parse_mode: 'HTML'
        message: |
          {{ range .Alerts }}
          [{{ .Status | toUpper }}] {{ .Annotations.summary }}
          {{ .Annotations.description }}
          {{ end }}
```

- **bot_token** — токен от BotFather.
- **chat_id** — в кавычках (строка). Для канала/группы отрицательное число в кавычках.

### 4.2 Направить алерты в Telegram

В секции `route:` добавьте дочерний маршрут, чтобы критичные (и при желании предупреждения) шли в Telegram. По правилам Pigsty у критичных алертов `level="0"` и `severity: CRIT`:

```yaml
route:
  receiver: 'default'
  group_by: ['alertname', 'instance']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 4h

  routes:
    # Критичные алерты — в Telegram
    - receiver: 'telegram'
      matchers: [ level="0" ]
      group_wait: 10s
      repeat_interval: 1h

    # Опционально: предупреждения тоже в Telegram
    # - receiver: 'telegram'
    #   matchers: [ level="1" ]
    #   repeat_interval: 4h
```

Если в ваших правилах используется `severity: CRIT` (а не `level="0"`), замените на:

```yaml
    - receiver: 'telegram'
      matchers: [ severity="CRIT" ]
```

### 4.3 Применить конфиг

На сервере Pigsty:

```bash
sudo systemctl reload alertmanager
# или
sudo systemctl restart alertmanager
```

Проверка: `systemctl status alertmanager` — должен быть active.

1. Временно сымитировать алерт (например отключить один из экспортеров или поднять порог в правиле) и убедиться, что сообщение пришло в Telegram.
2. Либо отправить тестовое уведомление через Alertmanager API (если умеете) или дождаться реального срабатывания.

## 6. Кратко

| Вопрос | Ответ |
|--------|--------|
| Как включить алерты? | Правила уже работают (VMAlert). Включить = добавить в Alertmanager receiver и route. |
| Куда слать? | В Telegram — через бота (обязательно) и выбранный chat_id (личный чат, группа или канал). |
| Отдельный канал или бот? | Нужен **бот** для доставки. Канал — по желанию, как место, куда слать (удобно отдельный канал «Alerts»). |
| Где править? | `/etc/alertmanager.yml` на 104.223.25.234. После правок — `systemctl reload alertmanager`. |

После настройки критичные алерты (NodeDown, PostgresDown, диск, и т.д.) будут приходить в выбранный чат/канал в Telegram.
